Forms (Rails only)

Use the form_for helper to display Bootstrap-styled forms.

bootstrap_ui_helper engine override the original form_for and almost all input field helper methods.

Basic forms

Use form_for without options to add Bootstrap style to the form elements.

  
    <%= form_for User.new, url: new_user_session_path do |f| %>
  
      <%= f.text_field :email, placeholder: 'Email' %>
  
      <%= f.password_field :password, placeholder: 'Password' %>
  
      <%= f.check_box :remember_me, label: 'Remember me' %>
  
      <%= f.submit 'Sign in', type: 'primary' %>
  
    <% end %>

generates the following HTML:

EXAMPLE
  
    <form class="form" id="new_user" action="/users/sign_in" accept-charset="UTF-8" method="post">
      <div class="form-group">
        <label class="" for="user_email">Email</label>
        <input placeholder="Email" class="form-control" type="text" value="" name="user[email]" id="user_email">
      </div>
      <div class="form-group">
        <label class="" for="user_password">Password</label>
        <input placeholder="Password" class="form-control" type="password" name="user[password]" id="user_password">
      </div>
      <div class="checkbox">
        <label>
          <input name="user[remember_me]" type="hidden" value="0">
          <input type="checkbox" value="1" name="user[remember_me]" id="user_remember_me">Remember me</label>
      </div>
      <input type="submit" name="commit" value="Sign in" class="btn btn-primary">
    </form>

Horizontal forms

Use form_for with layout: :horizontal to align labels and groups of form controls in a horizontal layout. The ratio of label and input field length is 1:3(col-sm-3, col-sm-9).

  
    <%= form_for User.new, url: new_user_session_path, layout: :horizontal do |f| %>
  
      <%= f.text_field :email, placeholder: 'Email' %>
  
      <%= f.password_field :password, placeholder: 'Password' %>
  
      <%= f.check_box :remember_me, label: 'Remember me' %>
  
      <%= f.submit 'Sign in', type: 'primary' %>
  
    <% end %>

Inline forms

Use form_for with layout: :inline for left-aligned and inline-block controls.

  
    <%= form_for User.new, url: new_user_session_path, layout: :inline do |f| %>
  
      <%= f.text_field :email, placeholder: 'Email' %>
  
      <%= f.password_field :password, placeholder: 'Password' %>
  
      <%= f.check_box :remember_me, label: 'Remember me', checked: true %>
  
      <%= f.submit 'Sign in', type: 'primary' %>
  
    <% end %>

generates the following HTML:

EXAMPLE
  
    <form class="form form-inline" id="new_user" action="/users/sign_in" accept-charset="UTF-8" method="post">
      <div class="form-group">
        <label class="sr-only" for="user_email">Email</label>
        <input placeholder="Email" class="form-control" type="text" value="" name="user[email]" id="user_email">
      </div>
      <div class="form-group">
        <label class="sr-only" for="user_password">Password</label>
        <input placeholder="Password" class="form-control" type="password" name="user[password]" id="user_password">
      </div>
      <div class="checkbox">
        <label>
          <input name="user[remember_me]" type="hidden" value="0">
          <input type="checkbox" value="1" checked="checked" name="user[remember_me]" id="user_remember_me">Remember me
        </label>
      </div>
      <input type="submit" name="commit" value="Sign in" class="btn btn-primary">
    </form>

Labels

Field helpers inside bootstrapped forms are automatically displayed with their relative label expect for check_box and radio_button. Use the :label option if you need to customize the generated label.

  
    <%= form_for User.new, url: new_user_session_path, layout: :horizontal do |f| %>
  
      <%= f.text_field :email, label: 'Your Email' %>
  
      <%= f.password_field :password, label: false %>

Placeholders

Use the :placeholder option if you need to customize the generated placeholder.

  
    <%= form_for User.new, url: new_user_session_path, layout: :horizontal do |f| %>
  
      <%= f.password_field :password, placeholder: 'At least 8 characters' %>
  
    <% end %>

generates the following HTML:

EXAMPLE
  
    <form class="form form-horizontal" id="new_user" action="/users/sign_in" accept-charset="UTF-8" method="post">
      <div class="form-group">
        <label class="col-sm-3 control-label" for="user_password">Password</label>
        <div class="col-sm-9">
          <input placeholder="At least 8 characters" class="form-control" type="password" name="user[password]"
                 id="user_password">
        </div>
      </div>
    </form>

Input groups

Field helpers inside bootstrapped forms accept the :prefix and :suffix options. Their values are displayed before/after the field as input groups. You can use strings or icons.

  
    <%= form_for User.new, url: new_user_session_path, layout: :horizontal do |f| %>
  
      <%= f.text_field :email, prefix: {icon: :envelope}, suffix: {icon: :envelope} %>
  
      <%= f.text_field :amount, prefix: '$', suffix: '.00' %>
  
    <% end %>

generates the following HTML:

EXAMPLE
$ .00 
  
    <form class="form form-horizontal" id="new_user" action="/users/sign_in" accept-charset="UTF-8" method="post">
      <div class="form-group">
        <label class="col-sm-3 control-label" for="user_email">Email</label>
        <div class="col-sm-9">
          <div class="input-group">
            <span class="input-group-addon"><i class="fa fa-envelope"></i></span>
            <input class="form-control" type="text" value="" name="user[email]" id="user_email">
            <span class="input-group-addon"><i class="fa fa-envelope"></i></span>
          </div>
        </div>
      </div>
      <div class="form-group">
        <label class="col-sm-3 control-label" for="user_amount">Amount</label>
        <div class="col-sm-9">
          <div class="input-group">
            <span class="input-group-addon">$</span>
            <input class="form-control" type="text" name="user[amount]" id="user_amount">
            <span class="input-group-addon">.00</span>
          </div>
        </div>
      </div>
    </form>

Help text

Use help to display a help message after fields inside bootstrapped forms.

  
    <%= form_for User.new, url: new_user_session_path do |f| %>
  
      <%= f.text_field :email, help: 'We will send a confirmation email' %>
  
      <%= f.password_field :password, help: 'Do not use "password"' %>
  
      <%= f.check_box :remember_me, label: 'Remember me', help: 'Keep signed in for a week' %> %>
  
      <%= f.submit 'Sign in', type: 'primary' %>
  
    <% end %>

generates the following HTML:

Keep signed in for a week

Supported Controls

Inputs

Most common form control, text-based input fields helper methods are override to include bootstrap styled form inputs: :email_field, :password_field, :text_field, :text_area, :search_field, :telephone_field, :url_field, :number_field, :file_field, :date_field, :time_field, :month_field, :week_field, :datetime_field, :datetime_local_field

  
    <%= form_for User.new, url: new_user_session_path do |f| %>
  
      <%= f.email_field :email %>
  
      <%= f.password_field :password %>
  
      <%= f.text_field :first_name %>
  
      <%= f.telephone_field :phone %>
  
      <%= f.number_field :count %>
  
      <%= f.file_field :avatar %>
  
      <%= f.text_area :content %>
  
      <%= f.date_field :confirmed_at %>
  
      <%= f.datetime_local_field :locked_at %>
  
    <% end %>

generates the following HTML:

EXAMPLE
  
    <form class="form form-horizontal" id="new_user" enctype="multipart/form-data" action="/users/sign_in"
          accept-charset="UTF-8" method="post">
